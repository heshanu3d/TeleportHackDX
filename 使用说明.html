TeleportHackDX 使用说明 DirectX9 + ImGui 内嵌界面版 TeleportHack — 构建、注入与使用全流程

1. 这是什么

TeleportHackDX 是 TeleportHackOnVanilla（原 AutoIt 版本）的完整 C++/ImGui 移植版。 它不再是一个外部 AutoIt/Python 进程通过 ReadProcessMemory 远程读写 WoW 内存， 而是一个被注入进 WoW.exe 进程内部的 DLL：它 Hook 住 Direct3D9 的 EndScene / Reset，直接在游戏画面上绘制一个 Dear ImGui 界面， 和 AzDeltaQQ/WoW3.3.5aDirectXHookWithGui 参考项目使用相同的 Hook 方式。

支持的客户端版本：1.12.1（Vanilla）、1.12.3（Vanilla）、3.3.5a（WLK），均为 32 位客户端。

2. 准备工作（需要一台 Windows 机器）

这个 DLL 必须运行在真实的 Windows 进程内部（Hook DirectX9），无法在 Linux/Ubuntu 上运行， 也不建议用 Wine（Wine 下 DirectX9 Hook 和游戏客户端本身兼容性都不可靠）。你需要：
  • 一台 Windows 10/11 物理机，或者
  • 一台 Windows 虚拟机（VMware / VirtualBox / 云端 Windows VM 均可）
在这台机器上安装好目标版本的 WoW 客户端（32 位）。

3. 获取产物

项目现在会同时产出两个程序：TeleportHackDX.dll（注入版，用于真正在游戏里用） 和 TeleportHackDesktop.exe（独立桌面版，见 第 8 节，不需要注入，用来验证界面/收藏点/热键逻辑）。 它们共用同一套界面和业务逻辑源码。

方式 A：直接下载 CI 构建产物（推荐，无需装 Visual Studio）

  1. 打开仓库 Actions 页面： https://github.com/heshanu3d/TeleportHackDX/actions
  2. 点击最新一次状态为绿色 ✅ 的 build workflow run
  3. 页面下方 Artifacts 区域下载 TeleportHackDX-win32.zip
  4. 解压后得到：TeleportHackDX.dll、TeleportHackDesktop.exe、 favlist.fav、hotkey.txt

方式 B：本地用 Visual Studio 编译

git clone git@github.com:heshanu3d/TeleportHackDX.git
cd TeleportHackDX
mkdir build && cd build
cmake -A Win32 ..
cmake --build . --config Release
# 产物在 build\bin\Release\TeleportHackDX.dll
# 以及  build\bin\Release\TeleportHackDesktop.exe

需要 Visual Studio 2019+ 和 CMake 3.15+；必须编译为 32 位（Win32）， 因为 WoW 这几个版本客户端都是 32 位程序。不需要额外安装老版 DirectX SDK。

4. 整理文件目录

把下面这些文件放在同一个文件夹里（这个文件夹可以在磁盘任意位置， 不需要和 WoW.exe 放一起）： 

C:\WowHack\
├── TeleportHackDX.dll
├── favlist.fav
└── hotkey.txt
为什么必须放一起？因为这个 DLL 被注入进游戏进程后，没有常规意义上的"当前工作目录"。 代码里（src/util/paths.cpp）会在运行时反查"自己这个 DLL 文件在磁盘上的哪个目录"， 然后固定去这个目录下寻找 favlist.fav / hotkey.txt / settings.json。 也就是说这三个文件跟着 DLL 走，跟 WoW 装在哪里没有关系。

你的注入工具放这里

仓库里预留了 TeleportHackDX/dll_injector/ 目录，你可以把下载好的注入工具 （见下一节）放进去，方便和这个项目放在一起管理。

5. 注入到 WoW.exe

  1. 1启动 WoW 客户端（对应版本，32 位），进到角色选择界面或者进入游戏世界都可以 ——只要窗口和 D3D9 设备已经创建即可。
  2. 2选一个通用 DLL 注入工具（任选其一，放进 dll_injector/ 目录）：
    工具特点操作方式
    System Informer
    （原 Process Hacker，推荐）    		 免费开源，界面直观，日常最常用的进程工具之一 右键 WoW.exe 进程 → Miscellaneous → Inject DLL... → 选择 TeleportHackDX.dll
    Extreme Injector 专用注入器，界面简单，新手友好 选中 WoW 进程 → 浏览选择 DLL → 注入方式保持默认 LoadLibrary → 点击 Inject
    Xenos 功能较全的注入器 操作流程类似 Extreme Injector
    Cheat Engine 自带的注入功能也可以用，完整步骤见下方 5.1 使用 Cheat Engine 注入。
  3. 3注入成功后，游戏内按 Win + Insert 显示/隐藏悬浮界面。

5.1 使用 Cheat Engine 注入（详细步骤）

Cheat Engine 从 6.4 版本开始自带了通用的"Inject DLL"功能，本身就是一个成熟稳定的注入器， 不需要额外下载别的工具。以下是完整流程：
  1. 1下载安装 Cheat Engine：官网 cheatengine.org 下载最新版（7.x）， 安装时如果弹出捆绑软件推荐，一路选择跳过/拒绝即可。
  2. 2先启动 WoW 客户端（对应版本，32 位）， 进到角色选择界面或游戏世界，确保进程和窗口已经存在。
  3. 3启动 Cheat Engine。如果 Windows Defender / 杀毒软件弹出拦截提示（Cheat Engine 自带的驱动组件很容易被误报），需要允许运行/加入白名单， 否则后续注入会失败。
  4. 4选择目标进程：点击 Cheat Engine 主窗口左上角 那个"电脑显示器+放大镜"图标（Process List / Open Process 按钮）， 在弹出的进程列表里找到并双击 WoW.exe 完成附加（attach）。 如果列表里看不到 WoW.exe，确认第 2 步的游戏客户端确实已经在运行。
  5. 5打开 Memory Viewer：附加成功后， 点击主窗口工具栏里那个类似"内存条/二进制"的图标（或菜单里的 Memory View， 快捷键通常是 Ctrl+M），会打开一个新的 "Memory Viewer" 窗口。
  6. 6注入 DLL：在 Memory Viewer 窗口的菜单栏中， 找到 Tools → Inject dll（不同 CE 版本菜单名称可能是 "Inject dll" 或 "Inject DLL into process"），点击后弹出文件选择框，浏览选中你的 TeleportHackDX.dll（放在 dll_injector/ 或前面第 4 步整理的 C:\WowHack\ 目录里都行），点击"打开"确认注入。
  7. 7确认注入结果：正常情况下不会有任何弹窗提示"成功"， 这是正常现象。切回 WoW 游戏窗口，按 Win + Insert，应该能看到悬浮界面出现。 如果想再次确认 DLL 确实已加载，可以在 Cheat Engine 里查看 Memory View → Symbols / Modules 列表，里面应该能找到 TeleportHackDX.dll。
  8. 8后续可以关闭 Cheat Engine： DLL 一旦通过 LoadLibrary 方式注入成功，就已经常驻在 WoW 进程里了， 和 Cheat Engine 本身是否还开着无关，可以放心关掉 CE 窗口。
Cheat Engine 常见坑：
  • 进程列表里找不到 WoW.exe / 附加失败： 尝试以"管理员身份运行" Cheat Engine 再重新附加一次。
  • 32 位/64 位不匹配报错： 现代 Cheat Engine 安装包内置了 32 位和 64 位两套注入辅助程序，会根据目标进程位数自动选择， 一般不需要手动处理；如果确实报位数不匹配的错误，去官网重新下载最新完整安装包（而不是绿色版/旧版）。
  • 杀毒软件把 Cheat Engine 或注入后的 DLL 删除/隔离： 需要把 Cheat Engine 安装目录和 TeleportHackDX.dll 所在文件夹都加入杀毒软件的排除项， 这是外挂/内存工具类软件的通病，属于误报而非真的有恶意代码。
  • Memory Viewer 菜单里没有 "Inject dll" 选项： 说明这个 Cheat Engine 版本较老，去官网升级到最新版即可，"Inject dll" 从很早的版本开始就是标配功能。

6. 界面使用说明

界面布局完全对应原 AutoIt/Python 版本的窗口结构：

区域说明
client 下拉框选择当前客户端版本：1.12.1 / 1.12.3 / 3.3.5，默认 3.3.5，务必先选对再传送
分类 / 搜索按描述前缀（-之前的部分）分类筛选，搜索框按描述模糊匹配
传送点表格显示 描述/x/y/z/r（r=朝向，无朝向时显示"-"）；单击选中一行，双击直接传送
描述输入框新增/编辑传送点时填写描述文本
step 勾选框勾选后传送变为"分步平滑移动"，不勾选则瞬间传送
AntiJump / AutoLoot / LuaUnlock / PatchLoot四个内存补丁开关按钮，点一次切换一次状态，结果显示在下方日志里
插前 / 插后 / 编辑 / 删除基于当前选中行 + 描述输入框 + 当前读取到的角色坐标，插入/编辑/删除该分类下的传送点
追加 / 保存 / 重载追加=在列表末尾新增一条；保存=写回 favlist.fav；重载=从磁盘重新读取 favlist.fav + hotkey.txt
Teleport 大按钮传送到当前选中的传送点
Speed 行设置移动速度（仅 3.3.5 支持，1.12.x 灰色不可用）
Import / Export导入替换全部 / 合并去重导入 / 导出到指定 .fav 文件（弹出系统文件选择框）
Settings调整分步移动的步距/间隔、favlist.fav 和 hotkey.txt 的路径
日志区所有操作结果、错误信息都会打印在这里

另外，hotkey.txt 里配置的组合键（如 ^1 = Ctrl+1）无论悬浮界面是否显示都会生效， 这一点和原版 AutoIt 的全局热键行为一致。

7. 数据文件格式

favlist.fav

描述#x#y#z#r
斯坦索姆-入口#1234.5#-678.9#12.3#
斯坦索姆-出口贴脸#1234.6#-678.8#12.3#1.857

描述里 - 前面的部分会被当作分类名，比如上面第一行会归到"斯坦索姆"分类下。

第 5 个字段 r 是朝向（弧度），仅 3.3.5 客户端支持，1.12.1/1.12.3 没有对应的内存偏移，永远读不到也不会写入。r 为空（行尾直接是 #） 表示"不改变当前朝向"——这和显式写 r=0（朝向正北）是两回事，程序会区分对待， 包括去重合并时也会把"无朝向"和"朝向=0"当成不同的记录。旧的 4 字段格式（没有第 5 个 #） 仍然兼容，加载后朝向为空；执行"保存"后会统一规范化成 5 字段格式。 分步传送（勾选 step）时，朝向只会在最后一步生效，不会导致中途转向。

hotkey.txt

# 以 # 开头的行是注释
^1
斯坦索姆-入口
!a
斯坦索姆-出口

两行一组：第一行是组合键（^=Ctrl，!=Alt，+=Shift），第二行是要跳转到的传送点描述（必须和 favlist.fav 里的描述完全一致）。

settings.json

首次运行会自动生成在 DLL/EXE 同目录下，记录分步移动参数、上次选择的分类、速度值等，缺失或损坏时会自动回退到默认值，不会导致崩溃。

8. 桌面测试版（不用注入也能验证功能）

TeleportHackDesktop.exe 是一个普通的桌面窗口程序（不需要注入任何进程）， 和注入版 TeleportHackDX.dll 共用同一套界面 + 业务逻辑源码 （src/ui/app.cpp 等），唯一的区别是它自己开一个窗口/D3D9 设备来画界面， 并且通过 OpenProcess + ReadProcessMemory/WriteProcessMemory "远程"读写一个已经在运行的 WoW.exe，而不是被注入进去直接读写自己所在的进程。 这样你改界面逻辑、收藏点逻辑、热键逻辑的时候，双击运行这个 exe 就能验证，不用每次都重新注入。

使用方法

  1. 1正常启动 WoW 客户端（不需要注入任何东西）。
  2. 2双击运行 TeleportHackDesktop.exe （建议和 favlist.fav、hotkey.txt 放在同一目录，原因同第 4 节）。 会弹出一个独立窗口，标题是 "TeleportHack Desktop (Attach mode)"，界面顶部比注入版多了一块 "WoW 进程" 区域。
  3. 3在 "WoW 进程" 列表里应该能看到正在运行的 WoW.exe （按 PID 显示）；点一下选中，再点 Attach 按钮（或者直接双击那一行）完成附加。 附加成功后旁边状态会显示"attached (PID xxxx)"。
  4. 4选好客户端版本（client 下拉框），之后的操作和注入版完全一样： 分类/搜索/传送/CRUD/开关/速度/导入导出都是同一套代码，行为完全一致。
桌面版这个窗口本身不会跑在 WoW 进程里面，只是"隔着"读写它的内存，所以：
  • 不需要任何注入器；
  • 如果 WoW 客户端崩溃/重启，需要重新点一次 Attach（PID 变了）；
  • 它更适合开发调试、验证收藏点/热键/UI 逻辑，正式"上分"建议还是用注入版 TeleportHackDX.dll，体验和原版 AutoIt 工具的使用方式最接近。

9. 常见问题排查

  • 注入后没反应 / 按 Win+Insert 没有界面： 确认 DLL 真的被加载了（在 System Informer 里查看该进程的 Loaded Modules 列表里能不能找到 TeleportHackDX.dll）；确认游戏窗口正在用 D3D9 渲染（不是 D3D11/OpenGL 私服客户端）； 确认没被杀毒软件/Windows Defender 隔离（游戏外挂类工具很容易被误报，需要加白名单）。
  • 中文描述显示成方块/乱码： 程序运行时会自动从 C:\Windows\Fonts 加载微软雅黑/宋体等中文字体； 如果系统没有这些字体文件，会退回 ImGui 内置的纯 ASCII 字体，中文无法显示。
  • 点击 AntiJump/AutoLoot 等按钮提示 "unknown byte state"： 说明当前客户端在对应内存地址的字节内容和预期的两种已知状态（开/关）都不一致， 可能是客户端版本/补丁不完全匹配 src/domain/versions.cpp 里写死的偏移， 为了安全起见程序会拒绝写入而不是覆盖成未知内容。
  • Speed 按钮点了没反应： Speed 只在 3.3.5 客户端上支持，1.12.1/1.12.3 客户端本身没有这个可写字段。
  • 桌面版点 Attach 没反应 / 提示 "failed to attach"： 大概率是权限问题——把 TeleportHackDesktop.exe 右键"以管理员身份运行"， OpenProcess 需要和目标进程相当的权限才能拿到读写句柄。
